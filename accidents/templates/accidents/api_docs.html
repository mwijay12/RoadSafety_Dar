Rate Limiting

POST /api/accidents — max 5 requests per 60 seconds per IP. Returns 429 Too Many Requests when exceeded. All other endpoints are rate-limited only at the infrastructure level.

Versioning

All endpoints are available under both /api/ and /api/v1/ prefixes. The unversioned /api/ paths will continue to work indefinitely. Use /api/v1/ for stable versioned access.

Error Format

All errors return JSON: {"error": "error_code", "detail": "Human-readable message"} 

HTTP 400: {"error": "validation", "detail": {"lat": ["Latitude is required"]}}
HTTP 429: {"error": "rate_limited", "detail": "max 5 submissions per 60s"}
HTTP 501: {"error": "not_implemented", "detail": "Feature not available"}
